Drift between the parallel `buildFileTree` paths and the `templates/*` content generators kept producing packages that diverged from the standard bearer (e.g. types/styles files weren't prefixed with the component name). New packages should be created by copying the standard-bearer layout (`nice-react-typography`) and appending a registry entry to `nice-toolkit/registry.json` by hand. Removed: `src/creator/`, `src/shared/registry/write.js` (only the creator called `addPackage`), the `--create` / `--type` parse-args entries, the usage banner lines, the dispatch in `src/index.js`. Option keys `create` and `createType` no longer parse.
